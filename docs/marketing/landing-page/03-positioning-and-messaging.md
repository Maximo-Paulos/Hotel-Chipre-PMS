# Positioning and Messaging

## ICPs por intencion de busqueda

### 1. "PMS hotelero"

Buyer intent:

- alta intencion
- sabe lo que busca
- probablemente compara software profesional

Pains:

- sistema actual corto o inexistente
- demasiadas tareas operativas repartidas
- falta de control diario

Mensaje recomendado:

- producto profesional
- centralizacion operativa
- onboarding claro

### 2. "software para hoteles" / "sistema de gestion hotelera"

Buyer intent:

- media/alta
- busca categoria, no necesariamente ya decidio PMS

Pains:

- herramientas desconectadas
- operacion manual
- poca visibilidad

Mensaje recomendado:

- explicar que cubre el sistema
- mostrar pantallas reales
- bajar friccion con trial

### 3. "administrar hotel" / "programa para hotel"

Buyer intent:

- mas amplio, mas pedagogico
- puede venir desde gestion manual

Pains:

- improvisacion
- reservas por varios canales
- cobros desordenados

Mensaje recomendado:

- lenguaje menos tecnico
- foco en "ordenar la operacion"

### 4. "motor de reservas" / "channel manager hotel"

Buyer intent:

- muy especifico
- peligro de prometer mas de lo verificable

Recomendacion:

- no usar estas keywords como promesa central de home hoy
- tratarlas como expansion SEO futura o menciones secundarias solo si el contenido visible y funcional lo soporta

### 5. "software hotel pequeno / boutique / independiente"

Buyer intent:

- muy alineado al target documentado

Mensaje recomendado:

- hotel independiente
- boutique
- pequeno/mediano
- sistema en espanol
- operacion centralizada

### 6. "alternativa a Excel / planillas / WhatsApp / gestion manual"

Buyer intent:

- problema muy concreto
- alto potencial de conversion si se lo trata bien

Tipo de claim:

- hipotesis de posicionamiento valida
- no usar como dato cuantificado

Mensaje recomendado:

- "deja atras el manejo disperso"
- "centraliza tu operacion"

## Posicionamiento principal recomendado

"Hotel Chipre PMS es un sistema de gestion hotelera para hoteles independientes que centraliza reservas, habitaciones, huespedes y cobros en un solo lugar."

Por que este posicionamiento:

- sale del producto real
- coincide con el lenguaje de busqueda
- no depende de claims no verificados
- permite crecer luego hacia funciones e integraciones sin romper la promesa base

## Tono de marca recomendado

- claro
- profesional
- directo
- sin humo
- operativo
- cercano al lenguaje del hotelero

Evitar:

- techno-hype
- promesas absolutas
- tono enterprise global si la salida es Argentina first
- claims de marketing sin evidencia

## Mensajes a validar luego

Estas ideas son utiles, pero hoy deben tratarse como hipotesis:

- "mas rapido que trabajar con Excel"
- "menos errores operativos"
- "mas control sobre disponibilidad"
- "menos friccion entre reservas y cobros"

Todas son plausibles, pero necesitan evidencia de uso, feedback real o medicion antes de convertirse en claim firme.
